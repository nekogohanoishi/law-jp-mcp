## 民法第709条（不法行為による損害賠償）

> **故意又は過失によって他人の権利又は法律上保護される利益を侵害した者は、これによって生じた損害を賠償する責任を負う。**

### 要件（4つ）

1. **故意又は過失** — 加害者に故意・過失があること
2. **権利・利益侵害** — 他人の権利又は法律上保護される利益を侵害したこと
3. **損害の発生** — 損害が生じたこと
4. **因果関係** — 侵害行為と損害との間に因果関係があること

### ポイント

- 日本の不法行為法の**一般的不法行為**の基本条文
- 立証責任は**被害者側**にある（過失責任の原則）
- 「法律上保護される利益」の文言は2004年（平成16年）改正で追加され、権利侵害だけでなく広く利益侵害も対象となることが明文化された

## 刑事訴訟法 — 公判前整理手続における証拠開示制度

裁判員裁判では**公判前整理手続が必須**（裁判員法49条）であり、その中で**三段階の証拠開示**が行われます。以下が関連条文です。

---

### 第1段階：検察官請求証拠の開示

#### 第316条の13（検察官の証明予定事実の明示）
> 検察官は、事件が公判前整理手続に付されたときは、その証明予定事実を記載した書面を、裁判所に提出し、及び被告人又は弁護人に送付しなければならない。

#### 第316条の14（検察官請求証拠の開示）
> 検察官は、前条の書面を提出した後、証拠の取調べを請求するについては、速やかに、被告人又は弁護人に対し、次の各号に掲げる証拠の区分に応じ、当該各号に定める方法による開示をしなければならない。
>
> 一　証拠書類又は証拠物　当該証拠書類又は証拠物を閲覧する機会を与えること（証拠物については、閲覧に代えて、謄写の機会を与えることができる。）
>
> 二　証人、鑑定人、通訳人又は翻訳人　その氏名及び住居を知る機会を与え、かつ、その者の供述録取書等のうち、その者が公判期日において供述すると思料する内容が明らかになるものを閲覧する機会を与えること

---

### 第2段階：類型証拠の開示

#### 第316条の15（類型証拠開示）
被告人・弁護人は、検察官請求証拠の開示後、検察官が保管する証拠のうち以下の**類型**に該当するものの開示を請求できる。

> **第1項各号の類型（主なもの）：**
>
> 一　証拠物
>
> 二　裁判所又は捜査機関の検証の結果を記載した書面
>
> 三　鑑定の経過及び結果を記載した書面（鑑定書）
>
> 四　検察官請求証人等の供述録取書等で、検察官が開示したもの以外のもの
>
> 五　検察官請求証人等の供述録取書等以外の者の供述録取書等であって、検察官が証明しようとする事実の有無に関する供述を内容とするもの
>
> 六　被告人の供述録取書等
>
> 七　取調べ状況の記録に関する書面
>
> 八　検察官請求証拠により直接証明しようとする事実の有無に関する書面であって、捜査の過程で作成されたもの

開示の要否は、**当該証拠の重要性の程度**と**開示によって生じるおそれのある弊害の内容・程度**を考慮して判断される（相当性の要件）。

---

### 第3段階：主張関連証拠の開示

#### 第316条の17（被告人・弁護人の主張明示）
> 被告人又は弁護人は、第316条の15の規定による開示後、その証明予定事実その他の公判期日においてすることを予定している事実上及び法律上の主張があるときは、裁判所及び検察官に対し、これを明らかにしなければならない。

#### 第316条の20（主張関連証拠の開示）
> 被告人又は弁護人は、第316条の17の規定により明らかにした主張に関連すると認められる証拠について、検察官に対し、その開示を請求することができる。

検察官は、**当該主張との関連性の程度**と**開示による弊害の内容・程度**を考慮し、相当と認めるときは開示しなければならない。

---

### 証拠開示に関する裁定・制限

#### 第316条の25（裁定請求）
> 検察官、被告人又は弁護人が証拠の開示に関する裁定を請求したときは、裁判所は、相手方の意見を聴いた上で、決定をしなければならない。

裁判所が証拠開示の可否・範囲について裁定する手続を規定。開示命令に対しては**即時抗告**が可能（同条3項）。

#### 第316条の26（開示証拠の目的外使用禁止）
> 被告人又は弁護人は、検察官から開示された証拠の複製等を、当該被告事件又はこれに密接に関連する手続以外の目的で使用してはならない。

違反には**懲役又は罰金**の刑事罰あり（281条の4・281条の5）。

---

### 制度の全体像（三段階構造）

| 段階 | 条文 | 内容 | トリガー |
|------|------|------|----------|
| **第1段階** | 316条の14 | 検察官請求証拠の開示 | 検察官の証明予定事実明示後 |
| **第2段階** | 316条の15 | 類型証拠の開示 | 被告人側の請求 |
| **第3段階** | 316条の20 | 主張関連証拠の開示 | 被告人側の主張明示後 |

## 刑事訴訟法 — 公判前整理手続における証拠開示関連条文

> 位置: 第二編 第一審 > 第三章 公判 > 第二節 争点及び証拠の整理手続 > 第一款 公判前整理手続

### 【第1段階】検察官請求証拠の開示

**第316条の13**（証明予定事実の明示）

検察官は、事件が公判前整理手続に付されたときは、その**証明予定事実**を記載した書面を、裁判所に提出し、及び被告人又は弁護人に送付しなければならない。この場合においては、裁判所に偏見又は予断を生じさせるおそれのある事項を記載することができない。
- 2項：証明予定事実を証明するための証拠の取調べを請求しなければならない
- 4項：裁判所は書面提出・請求の期限を定める

**第316条の14**（検察官請求証拠の開示）

検察官は、取調べを請求した証拠（**検察官請求証拠**）について、速やかに被告人・弁護人に開示しなければならない。
- **証拠書類・証拠物** → 閲覧の機会（弁護人には閲覧+謄写）
- **証人等** → 氏名・住居を知る機会＋供述録取書等の閲覧
- 2項：開示後、請求があれば**証拠一覧表**を交付しなければならない（2024年改正で追加）
- 4項：一覧表の記載除外事由（身体・財産への害、名誉毀損、捜査への支障のおそれ）

---

### 【第2段階】類型証拠の開示

**第316条の15**（類型証拠開示）

検察官請求証拠以外で、以下の**類型**に該当し、特定の検察官請求証拠の**証明力判断に重要**なものについて、開示請求があった場合、**必要性の程度**と**弊害の内容・程度**を考慮し、相当と認めるときは開示しなければならない。

| 号 | 類型 |
|----|------|
| 一 | 証拠物 |
| 二 | 裁判所・裁判官の検証結果書面 |
| 三 | 321条3項の書面（鑑定書等） |
| 四 | 321条4項の書面（実況見分調書等） |
| 五 | 検察官請求証人等の供述録取書等 |
| 六 | 被告人以外の者の供述録取書等（証明予定事実に関する供述） |
| 七 | 被告人の供述録取書等 |
| 八 | 取調べ状況記録書面 |
| 九 | 検察官請求証拠物の押収手続記録書面 |

---

### 【第3段階】主張関連証拠の開示

**第316条の17**（被告人側の主張明示）

被告人・弁護人は、第1段階・第2段階の開示を受けた後、**証明予定事実その他の事実上・法律上の主張**があるときは、裁判所及び検察官に対しこれを明らかにしなければならない。

**第316条の20**（主張関連証拠の開示）

被告人・弁護人の主張に**関連すると認められる証拠**について、開示請求があった場合、**関連性の程度**と**弊害の内容・程度**を考慮し、相当と認めるときは開示しなければならない。

---

### 証拠開示に関する裁定

**第316条の25**（開示の時期・方法の指定）

裁判所は、開示の必要性と弊害を考慮し、証拠開示の**時期・方法の指定**または**条件の付与**ができる。決定に対しては**即時抗告**可。

**第316条の26**（開示命令）

裁判所は、開示すべき証拠が開示されていないと認めるときは、相手方の請求により**証拠開示を命じなければならない**。決定に対しては**即時抗告**可。

## 各ライブラリの役割

### `law-aliases.ts` — 略称辞書
主要52法令の**略称→法令ID**マッピング。APIの検索ラウンドトリップを省略する。
```
民法 → 129AC0000000089
刑訴法 → 323AC0000000131
道交法 → 335AC0000000105
```

### `api-client.ts` — e-Gov APIクライアント
4つのエンドポイントに対応：
- `GET /laws` — 法令名検索
- `GET /law_data/{id}` — 法令全文取得（JSONツリー）
- `GET /keyword` — キーワード全文検索
- `GET /law_revisions/{id}` — 改正履歴

### `article-extractor.ts` — 条文抽出
e-GovのJSONは `LawNode` のツリー構造（HTMLのDOMに近い）。ここで：
- **漢数字→アラビア数字変換**（第三百十六条の十五 → `316_15`）
- **再帰的にArticleノードを探索**して該当条文を特定
- 条文の**編・章・節の位置情報**も `buildContext()` で取得

### `law-text-formatter.ts` — テキスト整形
e-GovのJSONツリーをLLMが読みやすいMarkdown風テキストに変換。構造的なタグ（Part, Chapter, Section, Article, Paragraph, Item...）を再帰的に処理する。

## 現状の課題

```
ユーザー: 「証拠開示の条文全部教えて」

現状のフロー:
  LLM: (脳内で316の13〜26を列挙)
  → get_article("316の13")  ← 1回目のAPI呼び出し
  → get_article("316の14")  ← 2回目
  → get_article("316の15")  ← 3回目
  → ...7回繰り返し...

問題:
  ① LLMが条番号を正確に知っている前提（古い/間違ってる可能性）
  ② APIラウンドトリップが多い（7回）
  ③ 条番号の「抜け」が起きうる
```

## 改善案

### A. 構造単位で取得するツール追加（`get_section`）

法令は **編 > 章 > 節 > 款 > 目** の階層構造。条番号を個別指定するのではなく、「この節を丸ごと取得」とすれば1回で済む。

```
// 現状: 条番号を1つずつ指定
get_article(lawId="刑事訴訟法", article="316の13")
get_article(lawId="刑事訴訟法", article="316の14")
...

// 改善後: 構造単位で一括取得
get_section(lawId="刑事訴訟法", path="公判前整理手続/争点及び証拠の整理")
```

### B. 条文範囲で取得（`get_articles`）

```
get_articles(lawId="刑事訴訟法", from="316の13", to="316の27")
```

### C. 目次（TOC）ツール追加

LLMが構造を把握してからピンポイントで取りに行ける。

```
get_toc(lawId="刑事訴訟法")
→ 第二編 第一審
    第三章 公判
      第二節 争点及び証拠の整理手続
        第一款 公判前整理手続
          第一目 ... (316条の2〜316条の12)
          第二目 争点及び証拠の整理 (316条の13〜316条の27) ← これ
          第三目 証拠開示に関する裁定 (316条の25〜316条の27)
```

### D. キーワード検索の強化

現状の `search_by_keyword` は該当箇所の位置だけ返す。条文テキストも一緒に返せば2段階が1段階になる。
